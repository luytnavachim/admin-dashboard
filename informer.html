Informer API Sandbox

Proof-of-concept om te zien of we vanuit dit dashboard direct inkoopfacturen kunnen aanmaken in Informer. API-docs: api.informer.eu/docs.

Werkt via een Cloudflare Worker. Informer staat geen directe browser-calls toe (CORS), dus we routeren via een Worker die de API-key server-side bewaart. Setup-stappen staan bovenaan in worker.js; de Worker-config staat in wrangler.jsonc. Zelfde Cloudflare-deploy hoste deze pagina ook.
1. Base URL in sectie 1 = https://<jouw-worker>.workers.dev/api — vaak gewoon /api (zelfde origin als deze pagina).
2. Auth-methode = "Geen (proxy regelt auth)"
3. API-key in dit veld leeg laten — die staat als Secret in de Worker.

2 Verbinding testen + administratie ophalen

Doet een GET op het opgegeven endpoint. Default /administration/ — geeft details van de administratie die bij je API-key hoort.

3 Bestaande inkoopfacturen ophalen

GET op het inkoopfacturen-endpoint uit instellingen (default /invoices/purchase/). De API-key bepaalt al welke administratie wordt geraakt.

4 Test-inkoopfactuur aanmaken

POST met onderstaande JSON. Vul eerst de placeholder-IDs (op 0) in. Gebruik sectie 2 om /relations/, /vat/, /ledgers/ en /costs/ op te halen en kopieer de juiste IDs hierin. invoice_total moet gelijk zijn aan de som van amount + vat_amount over alle regels.

5 Outlook → Informer (de echte flow)

Log in op Microsoft, kies een factuur-mail. Dashboard: (1) bijlagen ophalen (PDF + eventueel UBL-XML), (2) factuurdata extraheren — UBL eerst, anders Claude leest de PDF via POST /extract, (3) leverancier matchen op de geëxtraheerde naam via GET /relations/, (4) factuur aanmaken in Informer met PDF als base64 + de geëxtraheerde regels/totalen, (5) mail verplaatsen naar Verwerkt.
